SICStreamCipher class

NOTE: the implementation of SIC/CTR mode of operation as a BlockCipher is done using a StreamCipherAsBlockCipher adapter (see file [package:cipher/adapters/stream_cipher_adapters.dart] for more info).

Implementation of SIC mode of operation as a StreamCipher. This implementation uses the IV as the initial nonce value and keeps incrementing it by 1 for every new block. The counter may overflow and rotate, and that would cause a two-time-pad error, but this is so unlikely to happen for usual block sizes that we don't check for that event. It is the responsibility of the caller to make sure the counter does not overflow.
